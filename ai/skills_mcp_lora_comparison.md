# Skills、MCP、LoRA 对比

## 一览表

| 维度 | Skills | MCP | LoRA |
|--------|--------|--------|--------|
| 本质 | 工作流 / 提示工程 | 工具调用协议 | 模型参数微调技术 |
| 所处层次 | Agent 层 | Tool 层 | Model 层 |
| 主要作用 | 告诉 AI「怎么做」 | 让 AI「能做什么」 | 让 AI「变成什么样」 |
| 是否修改模型 | ❌ 否 | ❌ 否 | ✅ 是 |
| 是否需要训练 | ❌ 否 | ❌ 否 | ✅ 是 |
| 是否能访问外部系统 | 间接（通过 MCP） | ✅ 可以 | ❌ 不可以 |
| 开发成本 | 低 | 中 | 高 |
| 维护成本 | 低 | 低 | 高 |
| 典型内容 | Prompt、SOP、流程规范 | API、数据库、GitHub、Jira | 专业知识、行业能力、风格 |
| 适用场景 | 规范工作流程 | 集成外部工具 | 定制模型能力 |

---

## 直观理解

可以把 AI 看成一个程序员：

| 对象 | 类比 |
|--------|--------|
| LoRA | 给程序员进行专业培训 |
| MCP | 给程序员配电脑和工具 |
| Skills | 给程序员发操作手册 |

---

## Skills

### 定义

Skills 是一种可复用的工作流能力包，用于让 AI 按照预定义流程完成任务。

### 解决的问题

告诉 AI：

> 应该按照什么步骤完成工作。

### 示例

Rust 项目发布流程：

```text
1. cargo test
2. cargo clippy
3. 更新 changelog
4. 创建 Release
```

以后只需：

```text
执行 release
```

AI 即可自动按照规范执行。

### 特点

优点：

- 开发简单
- 无需训练
- 易于维护
- 团队知识可沉淀

缺点：

- 不增加模型能力
- 无法直接访问外部系统

---

## MCP（Model Context Protocol）

### 定义

MCP 是一种标准协议，用于让 AI 调用外部工具和系统。

### 解决的问题

让 AI：

> 能够访问现实世界的数据和服务。

### 示例

通过 MCP 接入：

- GitHub
- Jira
- PostgreSQL
- Kubernetes
- Slack

AI 可以：

```text
查询 Issue
创建 PR
执行 SQL
触发部署
发送消息
```

### 特点

优点：

- 连接企业系统
- 获取实时数据
- 执行真实操作

缺点：

- 需要开发 MCP Server
- 需要权限管理

---

## LoRA（Low-Rank Adaptation）

### 定义

LoRA 是一种参数高效微调（PEFT）技术。

通过训练少量附加参数，让模型获得新的领域能力。

### 解决的问题

让 AI：

> 在某个专业领域表现得更好。

### 示例

训练医疗 LoRA：

```text
GPT
  ↓
LoRA 微调
  ↓
医疗 GPT
```

获得：

- 医学术语理解
- 病历格式生成
- 医疗问答能力增强

### 特点

优点：

- 专业能力提升明显
- 参数量小于全量微调

缺点：

- 需要训练数据
- 需要训练资源
- 更新成本较高

# 三者关系

现代 Agent 系统通常可以表示为：

```text
          ┌──────────┐
          │  LoRA    │
          │(模型能力)│
          └────┬─────┘
               │
               ▼
          ┌──────────┐
          │   LLM    │
          └────┬─────┘
               │
        ┌──────┴──────┐
        ▼             ▼
    Skills       MCP Tools
   (工作流)       (工具能力)
```

---

## 实际例子

用户：

```text
发布今天的 Rust 版本
```

执行过程：

```text
Skill
 ├─ 获取发布步骤
 ├─ 调用 GitHub MCP
 ├─ 调用 CI MCP
 ├─ 调用 Jira MCP
 └─ 生成 Release Notes
```

其中：

### LoRA

决定：

```text
是否擅长 Rust
是否了解项目领域知识
```

### Skill

决定：

```text
发布流程怎么走
遵循什么规范
```

### MCP

决定：

```text
能否访问 GitHub
能否触发 CI
能否创建 Release
```

# 企业级 Agent 推荐组合

| 需求 | 推荐方案 |
|--------|--------|
| 接入企业系统 | MCP |
| 固化企业流程 | Skills |
| 行业知识增强 | LoRA |
| 构建研发 Agent | Skills + MCP |
| 构建企业 Copilot | Skills + MCP |
| 构建领域大模型 | LoRA + Skills + MCP |

---

# 当前行业趋势

近年来（Codex、Claude Code、Cursor、OpenHands 等）更偏向：

```text
Skills + MCP
```

原因：

- 成本低
- 上线快
- 易维护
- 实时获取数据
- 不需要重新训练模型

LoRA 更多用于：

- 医疗
- 法律
- 金融
- 制造业
- 私有领域知识

等对专业能力要求较高的场景。

---

# 一句话总结

```text
LoRA = 让 AI 更聪明

MCP = 让 AI 有工具

Skills = 让 AI 按流程做事
```

构建企业级 Agent 时：

```text
LoRA 决定能力
Skills 决定流程
MCP 决定行动
```
